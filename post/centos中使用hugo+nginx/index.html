CentOS中使用hugo+nginx

目前在网上出现各种各样的部署博客的方式, 其中在全世界范围内受用wordpress的最多. 但是作为个人博客, 他的量级太过重. 还有就是出现一些静态生成工具, 如hexo hugo

hexo 优点在于他的主题生态比较完善, 非常多好看的主题在其社区支持非常好. 但是小白就对他的安装诟病, 还需要安装node.js 并且如果文章数目多的时候就会导致生成页面很慢
hugo使用go开发的, 在生成页面上是hexo的几倍, 并且最近在学习go语言, 就顺利的选择了hugo, 缺点就是你想找个好看的主题,会让你憋出内伤.
网上很多文章都是使用hugo+gthub的, 不过这个方法在国内访问的时候支持不太好. 还没法用CDN, 因为国内网站使用CDN都需要备案. 使用国外的CDN好多人推荐Cloudflare, 测试过以后, 表示放弃.

在这里使用Centos服务器上部署hugo+nginx hugo是使用go语言开发的一个静态网页生成工具, 加载速度非常快. nginx对静态页面的支持尤其迅速, 二者结合如虎添翼, 并且不需要使用多高配置的服务器就可以达到效果. 可以进行备案域名, 国内使用CDN

安装hugo
yum install hugo -y

查看hugo版本, 有的主题要求hugo版本必须达到. 所以最好是能高一点
hugo version

2. 创建站点目录blog

我这里把站点目录放在data目录下
mkdir /data/
cd /data
hugo new site blog

3. 给hugo加载主题, 可以去官网下载.

由于官网的主题都是git包, 所以得先下载git
yum install git -y

进入主题目录
cd /data/themes
git clone https://github.com/xianmin/hugo-theme-jane.git jane
cd /data/blog/themes/inkblotty/exampleSite
cp config.toml /data/blog

将config.toml 复制到根目录下, 也就是/data/blog目录下, 目的是方便自己以后生成静态页面, 不用指定主题.
config.toml 可以由自己修改功能, 其中baseurl = “http://x.x.x.x" 这个地址是自己服务器的公网地址, 其他的参数在网上自己查找修改. 这里的使用http, 如果你的nginx配置ssl证书, 那么这里可以使用https

4. 生成html文件

hugo –theme=jane –baseUrl=“http://x.x.x.x"
–theme 为指定主题
–baseUrl 指定自己服务器启动的地址
此时会生成public目录, 为自己的站点目录

5. 安装nginx

这里使用tengine, 由淘宝发起的web服务项目, 与nginx配置相同, 但是在有些方面做过一些优化. 性能稳定性都在天猫和淘宝实践过

yum -y install pcre-devel openssl openssl-devel
wget https://tengine.taobao.org/download/tengine-2.3.0.tar.gz   
tar xf tengine-2.3.0.tar.gz    
cd tengine-2.3.0   
./configure  --prefix=/data/nginx   
make && make install
这里的/data/nginx 是nginx安装位置

cd /data/nginx/conf   
cat nginx.conf   

user  root;   
worker_processes  1;   
events {   
    worker_connections  1024;   
}   
http {   
    include            mime.types;   
    default_type       application/octet-stream;   
    sendfile           on;   
    keepalive_timeout  65;   
    gzip               on;   
    server {   
        listen       80;        
        server_name  x.x.x.x;   #服务器的地址   
        location / {   
            root   /data/blog/public;   #自己的站点目录   
            index  index.html index.htm;   
        }   
    }   
}
